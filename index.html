Operating runbook

How I ship.

The rules every one of my systems is built to — written the way an SOP would write them.

RULE-01
Happy path first, then armor.
Build the flow end to end, then layer in waits, retry loops and fallback selectors for every edge case.
RULE-02
Reliability over elegance.
A boring script that never fails beats a clever one that fails silently. Silent failures get hunted and surfaced.
RULE-03
Every step logs.
Status + timestamp written back to the sheet on every action — failures are traceable by design.
RULE-04
Surgical fixes, never rewrites.
Targeted find-and-replace over full-file rewrites. Production systems earn their scars; respect them.
RULE-05
Verify before you build.
Logic is checked against sample tickets and Metabase data before a line of automation is written.
RULE-06
Templated outputs only.
Freshdesk replies use verbatim templates; reports ship as .docx to Slack. Consistency is a feature.
